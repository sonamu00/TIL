## java.lang 패키지란
- 자파에서 import 없이 클래스를 사용할 수 있음
- 자바에서 꼭 필요한 여러 기능을 제공
- 꼭 알아야 하는 java.lang 예외
	- StatckOverflow
	- OutOfMemoryError

## 숫자를 처리하는 참조 자료형
- Byte, Short, Integer, Long, Float, Double => 기본 자료형에서 앞글자만 대문자 
- 기본 자료형을 숫자를 객체로 처리해야 할 필요가 있을 때 사용
- 숫자를 처리하는 클래스들은 `Number` 라는 `abstract` 클래스를 확장함
- 만든 이유
	- 매개변수를 참조 자료형으로 받는 메서드 처리
	- 기본 자료형을 사용하지 않는 기능을 사용하기 위해 => 제네릭
	- 진수 변환/파싱 API를 래퍼클래스에서 제공함 => 개발자가 직접 진수변환이나 문자 매핑 구현 로직을 구현하지 않아도 됨
	- 타입의 최소값과 최대값을 상수로 사용하기 위해서 => 코드에서 타입 범위 체크를 할 때 상수만 입력해도 됨(기본 자료형은 실제 범위값 입력해야함)

## System 클래스
- 시스템에 대한 정보를 확인하는 클래스
- 역할:
	- 시스템 속성 값 관리
		- 자바 프로그램의 시스템 속성값 조회 및 변경 가능
		- `java.util.Properties` 클래스의 객체로 값 관리 및 리턴
	- 시스템 환경 값 조회
		- OS나 장비의 환경값을 조회만 할 수 있음
	- GC 수행
		- JVM이 알아서 GC 처리 잘하기 때문에 쓰지말자
	- JVM 종료
	- 현재 시간 조회
	- 기타 관리용 메서드
- 3개의 static 변수가 스트림 클래스 타입으로 선언 되어 있음:
	- static PrintStream err; => 에러 출력 처리
	- static InputStream in; => 입력값 처리
	- static PrintStream out; => 출력값 처리
	=> 모두 static으로 선언되어 있어서 `Println()` 같은 메서드를 객체 생성없이 사용 가능함
