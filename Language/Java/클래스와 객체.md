## 클래스와 객체를 이해해야 되는 이유
1. 자바는 객체지향 언어여서 클래스와 객체 중심으로 프로그램이 구성된다.
2. 클래스와 객체를 사용해서 데이터와 데이터를 처리하는 메소드를 하나의 단위로 묶어 관리하기 때문이다.
3. 현실에서의 사물과 현상을 클래스와 객체로 표현하기 때문이다.

## 클래스와 객체의 차이점
- 클래스는 상태와 행동을 정의한다.
- 객체는 클래스에 정의된 상태와 행동을 실제 프로그램에서 구현한다.
- 클래스는 '정의', '관념', '설계도' 키워드로 나타낼 수 있다.
- 객체는 '실체', '실제 사물', '실제 행동', '프로그램 주체' 로 나타낼 수 있다.
##### 내 생각
클래스는 관념과 같아서 사람마다 생각하는 것이 다르다. 예를 들어 자동차라는 것을 생각했을 때, 어떤 사람은 몬스터 트럭을 생각할 수 있고 일반 세단 자동차를 생각할 수 있다. 어쩌면 뜬금없이 말을 생각하는 사람도 있을 수도 있다.
대전제는 대부분 같으나, 소전제로 넘어갈수록 사람마다 생각이 많이 다를 것이다. 그래서 개발 단계에서 클래스의 상태와 행위를 사람들과 소통하면서 세세하게 정의할 필요가 크다.

## 클래스의 구성 요소
1. 변수(속성): 객체의 상태를 저장할 때 사용, 객체의 상태 및 속성을 나타냄
2. 메서드(행위): 객체가 수행할 수 있는 작업을 정의할 때 사용, 객체의 행위를 나타냄
3. 생성자: 객체를 생성할 때 초기화 작업을 수행하는 메서드, 객체를 실제로 생성하는 역할을 한다.

## 클래스 컴파일 과정
1. 자바 컴파일러(javac)가 .java 파일 전체를 한 번에 읽는다.
2. 코드들을 최소 단위(ex: 키워드 - public, 식별자 - main, 연산자, 세미콜론)로 분리하고 문법 구조에 따라 추상 구문 트리 구조(AST)로 만든다.
3. 생성된 AST를 순회하며 코드의 문법적 오류와 의미적 오류를 검사한다.
	- 해당 단계에서 기본 생성자 자동 삽입이나 상수 폴딩 같은 간단한 코드 최적화가 처리됨
4. 오류가 없으면 JVM이 이해할 수 있는 코드인 바이트코드(.class) 파일을 생성한다.
=> 이후 해당 클래스 파일로 JVM에서 해당 OS/CPU의 기계어로 변환 및 실행
