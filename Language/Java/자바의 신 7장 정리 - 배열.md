## 1. 한 줄 요약
- 배열은 같은 타입의 여러 데이터를 하나의 변수 이름으로, 힙의 연속된 메모리 블록에 저장하는 자료구조이며, 자바에서는 안전한 인덱스 체크와 length 필드, 향상된 for문으로 편리하게 다룰 수 있다.

## 2. 핵심 개념 정리
### 2-1. 배열이란?
- 배열은 한 가지 타입에 대해 하나의 변수에 여러 개의 데이터를 순서대로 저장할 수 있는 가장 기본적인 자료구조이다.
- 배열의 각 저장 위치를 요소(element)라고 하며, 요소를 구분하기 위해 0부터 시작하는 인덱스(index)를 사용한다.

### 2-2. 배열 선언과 생성
- 배열 선언 형식: `자료형[] 변수명;` 또는 `자료형 변수명[];` 
	- 예: `int[] scores;`, `int scores[];`
- 선언 후 생성: `scores = new int[5];` 처럼 `new`와 크기를 사용해 힙 영역에 배열 객체를 생성한다.
- 선언과 생성을 한 번에: `int[] scores = new int[5];`
- 중괄호 초기화: `int[] nums = {1, 2, 3};` 처럼 값을 나열해 초기화할 수 있고, 이때 중괄호 뒤에는 세미콜론(;)을 붙인다.

### 2-3. 배열의 인덱스, 기본값, 예외
- 배열의 첫 번째 요소 인덱스는 0이고 마지막 인덱스는 `length - 1`이다.
- 크기만 지정한 배열은 타입별 기본값으로 자동 초기화된다.
	- 정수형 배열(`int[]`)의 요소 기본값은 0이다.
	- 논리형 배열(`boolean[]`)의 요소 기본값은 `false`이다.
	- 참조형 배열(`String[]` 등)의 요소 기본값은 `null`이다.
- 선언된 크기를 넘어선 인덱스로 접근하면 `ArrayIndexOutOfBoundsException`이 발생하며, 이는 잘못된 메모리 접근을 막기 위한 자바의 안전장치이다.

### 2-4. 배열의 메모리 구조(JVM, OS 관점)
- `int[] nums = new int[3];`와 같이 배열을 생성하면 실제 요소들이 저장되는 배열 객체는 JVM 힙(Heap) 영역에 연속된 메모리 블록으로 생성된다.
- 지역 변수 `nums`는 스택(Stack) 영역에 생성되며, 힙에 있는 배열 객체의 참조값(주소)을 저장한다.
- CPU는 "배열 시작 주소 + 인덱스 × 요소 크기"로 요소의 실제 메모리 위치를 계산해 빠르게 접근할 수 있고, 자바는 이 과정에서 인덱스 범위를 검사해 잘못된 접근을 예외로 막는다.

### 2-5. 2차원 배열
- 2차원 배열은 "배열을 요소로 가지는 배열"로, 표(행과 열) 형태의 데이터를 표현할 때 사용한다.
- 선언과 생성 예시: `int[][] table = new int[3][4];` (3행 4열)
- 리터럴 초기화 예시:
```java
  int[][] map = {
      {1, 2, 3},
      {4, 5, 6}
  };
```
- `table.length`는 행(바깥 배열)의 길이, `table[0].length`는 열(각 행 배열)의 길이를 나타낸다.

### 2-6. 배열을 쉽게 도는 향상된 for문
- 향상된 for문 기본 문법: `for (타입 임시변수 : 배열또는컬렉션) { ... }`
- 예시:
```java
int[] scores = {10, 20, 30};
for (int score : scores) {
    System.out.println(score);
}
```
- 인덱스가 필요 없고 모든 요소를 한 번씩 순회할 때 사용하면 코드가 간결해지고 인덱스 실수를 줄일 수 있다.
- 요소의 인덱스를 이용해야 하거나 요소를 교체해야 할 때는 기존 `for (int i = 0; i < 배열.length; i++)` 형태가 더 적합하다.
    
### 2-7. 배열 vs ArrayList 간단 비교(실무 관점)
- 배열(`int[]` 등)은 크기가 고정된 연속된 메모리 블록으로, 요소 개수가 고정되어 있고 인덱스 기반 랜덤 접근이 매우 빠른 경우에 적합하다.
- `ArrayList<T>`는 내부적으로 배열을 사용하지만 크기를 자동으로 늘려 주는 가변 리스트로, 요소 개수를 미리 알기 어렵고 추가와 삭제가 자주 일어나는 경우에 적합하다.
- 성능과 메모리 효율이 중요한 고정 크기 데이터(예: 24시간 에러 카운트 등)는 배열을, 크기와 내용이 자주 바뀌는 동적 목록(예: 검색 결과 리스트 등)은 ArrayList를 사용하는 것이 일반적이다.
    

## 3. 정리해봅시다(7장 연습문제 요약)
1. Q. 배열을 선언할 때 어떤 기호를 타입 또는 변수 이름의 앞이나 뒤에 사용해야 하나요?
	- A. 대괄호 `[]`를 사용한다. 예) `int[] arr;`, `int arr[];`
2. Q. 배열의 첫 번째 위치 인덱스는 0인가요, 1인가요?
	- A. 0부터 시작한다.
3. Q. boolean 배열의 크기만 지정하고 값을 넣지 않았을 때, `[0]` 위치의 기본값은 무엇인가요?
	- A. `false`가 저장된다.
4. Q. `ArrayIndexOutOfBoundsException`은 언제 발생하나요?
	- A. 선언한 배열의 크기를 벗어나는 인덱스로 접근하거나 값을 할당하려고 할 때 발생한다.
5. Q. 중괄호를 이용해 배열을 초기화할 때, 중괄호 끝에 반드시 어떤 기호를 붙여야 하나요?
	- A. 세미콜론 `;`을 붙여 문장을 끝낸다. 예) `int[] nums = {1, 2, 3};`
6. Q. 2차원 배열을 정의할 때에는 대괄호를 몇 개 지정해야 하나요?
	- A. 대괄호를 두 개 지정한다. 예) `int[][] table = new int[3][4];`
7. Q. 배열을 쉽게 처리해 주는 for문의 문법은 어떻게 되나요?
	- A. `for (타입이름 임시변수명 : 반복대상객체) { ... }` 형식으로 사용한다.
