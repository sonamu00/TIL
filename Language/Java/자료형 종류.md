### 8비트와 byte 타입
- 8비트는 0과 1로 표현할 수 있는 공간이 8개가 있는 것이고, 이 8비트 하나를 1바이트라고 한다.
- byte 공간은 8개인데 255 값이 아니고 127인 이유는 맨 앞자리는 양수 음수를 나타내기 위한 자리이다.
#### byte 의 최대값이 127, 최소값이 -128 인 이유
- 자바의 모든 정수형은 숫자를 표현할 때 **2의 보수** 방식을 사용한다.
- 2의보수는 1의보수와 다르게 0을 표현하는 방법이 하나(0000 0000)이기 때문에 양수, 음수, 0 이렇게 세가지의 숫자 조합만 가능하다.
- byte 는 표현할 수 있는 숫자가 총 256가지가 되는데 0이 하나만 존재해서 0을 제외한 숫자 범위는 총 255가지가 된다.

| 영역  |     | 범위 (8비트 byte 기준) | 개수   |
| --- | --- | ---------------- | ---- |
| 양수  |     | +1 ~ +127        | 127개 |
| 0   |     | 0                | 1개   |
| 음수  |     | -1 ~ -128        | 128개 |
 - ​**최대값**(+127)의 이진수: 0111 1111
 - **최소값**(-128)의 이진수: 1000 000
- 0이 하나뿐인 구조 덕분에 음수 범위가 -1 ~ -128 이 되면서 숫자를 하나 더 표현할 수 있게 되었다.
- byte 최소값(-128)에서 1을 빼면 `-128 - 1 = 127` , 최대값(127)에서 1을 더하면 `127 + 1 = -128` 이런 결과가 나오는 이유는 2의보수에선 부호비트 자리에서 자리올림이 발생하면 오버플로우가 발생한다.
- 오버플로우가 발생하면 1의 보수와 달리 순환올림을 하지 않고 자리 올림수를 버린다.
- 2의 보수에서 1을 더하는 이유는 값을 의도적으로 오버플로우를 발생시켜서 값을 순환하기 위함이다.
