## getClass() 메서드란
- 런타임에 객체의 클래스 정보를 리턴하는 메서드
- 런타임 실제 클래스를 확인할 수 있는 메서드
- 사용하는 곳:
	- instanceof 메서드로 하위 타입까지 체크할 필요가 없고, 정확히 같은 클래스만 비교할 때
	- 프레임워크/라이브러리 => 런타임 메타정보 조회
	- 리플렉션 진입점
	- 런타임에 어노테이션 읽기
	- 플러그인/동적 로딩 => ClassLoader/ServiceLoader와 같이 사용되는 경우가 많음
- 반환값: `java.lang.Class` 객체
	- 반환되는 `Class` 객체가 `static synchronized` 메서드로 잠그는(lock) 대상임

## `Class` 객체
- 런타임에서 클래스 자체를 대표하는 객체
- "이 객체/타입이 어떤 클래스인지"를 자바 코드에서 다루게 해주는 타입의 대표 객체
- 생성되는 시점:
	- public 생성자가 없고, JVM이 **클래스를 로딩할 때**(또는 클래스 로더의 `defineClass` 호출로 클래스를 정의할 때) `Class` 객체를 만듬
- `static synchronized` 로 `Class` 객체 전체에 락을 건다. => 클래스 락, 클래스 레벨 락, 모니터(JVM 스펙 용어)라고도 불림

### static synchronized
- `synchronized` 는 항상 메서드가 속해 있는 인스턴스를 잠금
- static(정적) 메서드면 잠금 대상 인스턴스가 없기 때문에(static은 메서드 영역) **메서드가 속한 클래스의 `Class` 객체를 잠금**
