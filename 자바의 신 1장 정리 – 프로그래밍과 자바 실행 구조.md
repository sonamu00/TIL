## 1. 프로그래밍이란 무엇인가
- 프로그래밍이란 
	-  사람의 목적(하고 싶은 일)을 **컴퓨터가 실행할 수 있는 형태(알고리즘 + 코드)** 로 설계하고 표현하는 과정
	- 단순 번역이 아니라:
		- 어떤 순서로 처리할지 (알고리즘)
		- 어떤 데이터를 쓸지 (자료 구조)
		- 어떤 조건에서 어떻게 동작할지 (분기/반복)
		를 **설계 + 코드로 표현**하는 것
	- 내 생각: 특정 나라의 언어로 얘기할 때 문화적 맥락을 고려하지 않으면 부정문이나 긍정문이 다르게 해석되는 경우가 있습니다. 프로그래밍도 마찬가지라고 생각합니다. 프로그래밍 언어가 어떻게 이렇게 구조가 완성되었는지, 어떠한 철학으로 개발 되었는지 모르면 프로그램이 제 의도와 다르게 해석되는 경우가 많겠죠. 따라서 프로그래밍은 사람의 언어를 단순 번역 하는 것이 아니고, 컴퓨터의 설계 구조와 발전하게 된 맥락을 이해해야 비로소 진정한 프로그래밍이라고 생각합니다.

## 2. 프로그램 vs 프로세스 vs 스레드

### 2.1 프로그램 (Program)
- 디스크(SSD/HDD)에 저장된, **아직 실행되지 않은 실행 가능한 코드와 데이터의 묶음**
- 예시: `myapp.jar`, `chrome.exe`, `SSEClient.exe`
- 상태적으로 보면 **정적인 설계도**
- 한 줄 정의:  **프로그램 = 디스크에 가만히 있는, 실행 가능한 코드 묶음**

### 2.2 프로세스 (Process)
- OS가 메모리와 각종 자원(파일 핸들, 스레드 등)을 붙여서 **실행 중인 프로그램의 인스턴스**
- 같은 프로그램이라도 여러 번 실행하면 **프로세스는 여러 개** 생김  
  - 예: `java -jar myapp.jar` 를 두 번 실행 → 프로그램 1개, 프로세스 2개
- 프로세스마다 **독립된 메모리 공간**을 가진다.
- 한 줄 정의:  **프로세스 = OS가 자원을 붙여 실행시키고 있는 프로그램 인스턴스**

### 2.3 스레드 (Thread)
- **CPU가 실제로 실행하는 제어 흐름의 최소 단위**
- 한 프로세스 안에 여러 스레드가 있을 수 있다.
- 각 스레드는 **자기만의 스택(Stack)** 을 가진다.  
    -  메서드 호출 정보, 지역 변수, 매개변수 등이 들어감
-  같은 프로세스 안의 스레드들은 **힙(Heap)과 메서드 영역(Method Area)을 공유**한다.  
    - 공유 객체, static 변수 등이 여기에 존재
- 한 줄 정의:  **스레드 = 프로세스 내부에서 자원을 공유하며 동작하는 CPU 실행의 최소 단위**

## 3. JVM, javac, .java, .class 관계

### 3.1 .java 파일
- 사람이 읽을 수 있는 **자바 소스 코드가 들어 있는 텍스트 파일**
- 자바 문법으로 작성됨
- **그 자체로는 실행되지 않고**, 반드시 자바 컴파일러(javac)로 컴파일해야 한다.
- .java = 사람이 작성/수정하는 소스 코드

### 3.2 .class 파일 (바이트코드)
- `javac`가 `.java` 파일을 컴파일해서 만든 **바이트코드 파일**
- CPU가 이해하는 기계어가 아니라, **JVM이 이해하는 중간 코드**
- 실행 시 JVM이 이 바이트코드를
	- 인터프리트하거나
	- JIT(Just-In-Time) 컴파일해서  
    OS/CPU가 이해하는 실제 기계어로 바꿔 실행한다.
-  `.class` → JVM이 읽어서 실행하는 바이트코드

### 3.3 컴파일 타임 오류 vs 런타임 오류
- **컴파일 타임 오류**
	- javac가 `.java`를 `.class`로 바꾸는 **컴파일 단계에서** 발생
	- 문법 오류, 타입 오류 등 코드만 보고 잡히는 오류
	- JVM의 classLoader 가 작동하기 전에만 발생할 수 있음
- **런타임 오류**
	- 프로그램이 실제로 실행된 뒤, **JVM이 바이트코드를 실행하는 도중** 발생
	- 예: `NullPointerException`, `ArrayIndexOutOfBoundsException`, DB 연결 실패 등
	- 실행을 해봐야 드러나는 오류
- 한 줄 정리  
	- 컴파일 타임 오류: 실행 전, **컴파일 단계에서** 잡히는 오류  
	- 런타임 오류: 실행 후, **실행 도중**에 터지는 오류

## 4. JVM의 역할
- JVM은 **`.class` 바이트코드를 읽어서** 해당 OS/CPU에서 실행되도록 함
	- 바이트코드를 해석(인터프리터)
	- JIT로 네이티브 코드로 컴파일
	- 메모리(힙/스택), 스레드, GC 등을 관리하는 **런타임 환경** 제공
- 한 줄 정의:  **JVM = 바이트코드를 실행하고, 메모리/스레드/GC를 관리하는 자바 실행 환경**

## 5. 자바 프로그램 실행 흐름 (요약 3단계)
1. **개발자가 .java 작성**  
	- 사람 눈에 보이는 소스 코드 작성
2. **javac로 컴파일**  
	- `.java` → `.class` (바이트코드)  
	- 이때 **컴파일 타임 오류**가 발생할 수 있음
3. **JVM으로 실행**  
   - `java` 명령으로 JVM 프로세스를 띄움  
   - JVM이 `.class` 를 로드하고, `main` 메서드부터 실행  
   - 인터프리터/JIT로 실제 CPU에서 돌아가게 하고, GC/스레드/메모리 관리 수행
